# StageOps

StageOps é uma plataforma em desenvolvimento para gestão operacional de empresas de eventos, com foco em locação de equipamentos de som, iluminação, painel de LED, palco, backline, estruturas e serviços técnicos.

O objetivo do projeto é centralizar processos que normalmente ficam espalhados em planilhas, mensagens, anotações e controles manuais, ajudando empresas de eventos a organizarem orçamentos, estoque, reservas por data, checklists de montagem e logística de carregamento e retorno.

## Problema que o StageOps busca resolver

Empresas de locação e produção de eventos lidam diariamente com desafios como:

* Controle de disponibilidade de equipamentos por data;
* Conflito de reservas entre eventos diferentes;
* Falta de padronização nos orçamentos;
* Dificuldade para acompanhar o que foi separado, carregado, montado e devolvido;
* Comunicação descentralizada entre comercial, logística e equipe técnica;
* Risco de esquecer cabos, acessórios, peças pequenas ou itens complementares;
* Falta de histórico organizado sobre clientes, eventos e operações realizadas.

O StageOps nasce para transformar esse fluxo em um sistema mais organizado, visual e confiável.

## Funcionalidades planejadas

### Gestão de equipamentos

* Cadastro de equipamentos por categoria;
* Controle de quantidade disponível;
* Organização por kits e conjuntos;
* Registro de itens como caixas de som, moving heads, refletores, painéis de LED, praticáveis, cabos, cases e acessórios.

### Orçamentos

* Criação de orçamentos para eventos;
* Seleção de equipamentos e serviços;
* Cálculo de valores;
* Registro de cliente, local, data e observações técnicas;
* Histórico de propostas enviadas.

### Reservas por data

* Verificação de disponibilidade dos equipamentos;
* Bloqueio de itens para eventos confirmados;
* Visualização de conflitos de agenda;
* Calendário operacional.

### Checklists operacionais

* Checklist de separação de equipamentos;
* Checklist de carregamento;
* Checklist de montagem;
* Checklist de desmontagem e retorno;
* Controle do que saiu e voltou para o estoque.

### Gestão de eventos

* Cadastro de eventos;
* Registro de informações do cliente;
* Local, horário, equipe e observações;
* Status do evento: orçamento, confirmado, em separação, em montagem, finalizado.

### Relatórios futuros

* Equipamentos mais utilizados;
* Eventos realizados por período;
* Receita estimada por evento;
* Histórico de clientes;
* Controle de perdas, avarias e manutenções.

## Como executar localmente

1. Clone o repositório:

```bash
git clone https://github.com/Novachrono117/StageOps.git
```

2. Entre na pasta do projeto:

```bash
cd StageOps
```

3. Abra o arquivo `index.html` no navegador.

Também é possível abrir a pasta no VS Code e utilizar uma extensão como Live Server para visualizar a página localmente.
